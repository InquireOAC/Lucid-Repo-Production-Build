

## Plan: Reseed Dream Content & Images

### Problem Summary
The current ~1,200 mock dreams all share the same templated story (`generateContent()`) that recycles phrases like "colors that had no name" and "rippling waves of light" regardless of the title. Many titles have "Revisited" / "Again" suffixes. Images were randomly assigned from a small pool of ~160 images that don't match dream content.

### Solution: New Edge Function `reseed-dream-content`

Create a single edge function with two phases, run in batches:

**Phase 1 — `content`**: For each mock dream (identified via user email `mock_*@lucidrepo-seed.test`):
- Strip "Revisited/Again/Continued/II/III" suffixes from titles
- Use Gemini 3 Flash to generate a unique 150-250 word dream story based on the specific title
- Update content in DB
- Also clear the old image_url so Phase 2 picks it up

**Phase 2 — `images`**: For each mock dream with no image:
- Use compose-cinematic-prompt to create a prompt from the new content
- Use Gemini 3.1 Flash Image to generate a matching 9:16 vertical image
- Upload to storage, update DB

Both phases accept a `limit` parameter for batch processing.

### Title Deduplication
Instead of appending "Revisited", dreams with duplicate titles will get a unique variation generated by the AI (e.g., "The Crystal Cathedral Under the Sea" stays once; duplicates get retitled by AI based on tags/mood).

### 9:16 Enforcement
The image generation prompt will include the vertical orientation directive at both the start and end of the content parts, matching the pattern in `generate-dream-image`.

### Files to Create/Modify
1. **Create** `supabase/functions/reseed-dream-content/index.ts` — new edge function for the reseed job
2. **Modify** `supabase/config.toml` — add function entry with `verify_jwt = false`

### Execution Plan
After deploying, I'll trigger multiple batch calls:
- Content: `{"phase": "content", "limit": 20}` repeated until all dreams are rewritten
- Images: `{"phase": "images", "limit": 5}` repeated until all dreams have matching images

### Technical Details
- Mock user identification: join `dream_entries` → `profiles` → `auth.users` via service role, filter by email pattern
- AI story generation: system prompt instructs first-person dream narrative, unique to title, no recycled phrases
- Image model: `google/gemini-3.1-flash-image-preview` (user's choice for speed)
- Wall clock: 300s limit for image batches
